# Step 4 创建第一个 Python 程序（Hello World）

> **目标：** 在 VS Code 中成功运行第一个 Python 程序，验证 Python 开发环境已经配置完成。

点击顶部菜单：


## 4.3 编写第一段 Python 代码

在编辑器中输入下面代码：


## 4.4 选择 Python 解释器

第一次使用 VS Code 时，需要告诉 VS Code 使用哪个 Python 环境。

```text

```

我建议这一部分一定要加，因为**Jupyter Notebook最大的特点就是"代码 + 文档"**，很多学生第一次接触Notebook，不知道Markdown Cell的用途。后面的实验报告、模型分析、AI实验记录都会用到Markdown。

可以把 **5.3** 修改成下面这样。

---

## 5.3 (1) 创建第一个代码单元（Code Cell）

Notebook 默认会创建一个空白的 **Code Cell（代码单元）**。

如果没有，可以点击页面顶部：

```text
+ Code
```

创建一个新的代码单元。

在第一个 Cell 中输入：

```python
print("Hello Notebook")
```

---

## 5.3 (2) 创建 Markdown 单元（Markdown Cell）

除了编写 Python 代码外，Jupyter Notebook 还支持编写实验说明、学习笔记和实验报告。

点击当前 Cell 上方的 **+** 按钮新增一个 Cell，然后在工具栏中将 **Code** 修改为：

```text
Markdown
```

或者使用快捷键：

```text
Esc

↓

M
```

即可将当前 Cell 切换为 **Markdown Cell**。

在 Markdown Cell 中输入下面内容：

```markdown
# My First Notebook

## AI Summer Camp

这是我的第一个 Jupyter Notebook。

- Python
- Jupyter Notebook
- Machine Learning
```

输入完成后，按：

```text
Shift + Enter
```

Notebook 会将 Markdown 渲染为格式化文档。

最终效果类似：

---

# My First Notebook

## AI Summer Camp

这是我的第一个 Jupyter Notebook。

* Python
* Jupyter Notebook
* Machine Learning

---

> **说明：**
>
> Markdown Cell 用于编写实验说明、标题、学习笔记、公式和结论，不会执行 Python 代码。后续课程中的实验报告、模型分析、结果总结等内容，建议都使用 Markdown Cell 编写，使 Notebook 同时具备代码和文档两种功能。

---

## 5.3 (3) Notebook 的基本结构

一个规范的 Notebook 通常由两种 Cell 组成：

| Cell 类型           | 用途                                |
| ----------------- | --------------------------------- |
| **Code Cell**     | 编写并运行 Python 代码，完成数据处理、模型训练和结果分析。 |
| **Markdown Cell** | 编写标题、实验说明、学习笔记、实验结论和文档说明。         |

建议后续所有实验 Notebook 都按照下面的结构组织：

```text
# 实验名称（Markdown）

实验目的（Markdown）

数据读取（Code）

数据分析（Code）

## 特征工程（Markdown）

特征工程代码（Code）

## 模型训练（Markdown）

训练代码（Code）

## 实验结果（Markdown）

结果分析（Markdown）
```

这样整理后的 Notebook 更接近大学课程实验和企业数据科学项目的规范，也方便后续直接上传到 GitHub 作为项目文档。

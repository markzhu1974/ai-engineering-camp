# Step 2：Understand（理解训练流程）

这段代码实现了一个基于 PyTorch 的经典机器学习流程：使用多层感知机（MLP）对 MNIST 手写数字数据集进行分类。

以下是该流程的核心步骤拆解：

## 1. 数据准备与预处理
下载与加载： 代码首先检查本地是否存在数据集，若不存在则自动下载 MNIST 数据集（包含 60,000 张 28x28 的训练图片）。

训练集加载器： 使用 DataLoader 将训练数据切分为大小为 50 的小批次（Mini-batch），并开启随机打乱（shuffle），方便进行批训练。

测试集截取： 提取了前 2,000 张测试图片，手动将其展平并除以 255 进行归一化（将像素值缩放到 0 到 1 之间），用于后续的性能评估。

## 2. 构建多层感知机（MLP）模型
定义了一个名为 MLP 的全连接神经网络，结构非常直接：

输入层到隐藏层： nn.Linear(28*28, 28*28)，将 784 维的展平图像输入到 784 个全连接神经元。

隐藏层到输出层： nn.Linear(28*28, 10)，最终输出 10 个类别的得分（对应数字 0-9）。

注：此处的网络未添加激活函数（如 ReLU），属于纯线性组合。

## 3. 定义损失函数与优化器
优化器： 选用 Adam 优化器，学习率（LR）设为 0.001，用于自动调整网络权重。

损失函数： 使用 交叉熵损失（CrossEntropyLoss），它非常适合多分类任务，且内部会自动对输出进行 Softmax 处理。

## 4. 模型训练与实时评估
程序仅训练 1 个 Epoch（将全量训练数据跑一遍），内部循环如下：

数据展平： 每次迭代读取 50 张图片，用 view(-1, 28*28) 将形如 (50, 1, 28, 28) 的四维张量展平为 (50, 784) 的二维张量。

前向传播与反向传播： 计算模型输出 -> 计算损失（Loss） -> 梯度清零（zero_grad） -> 反向传播计算梯度（backward） -> 更新参数（step）。

定期评估： 每隔 50 个批次（Step），使用前面准备好的 2000 张测试集图片对模型进行测试，计算并打印当前的训练损失（Loss）和测试准确率（Accuracy）。

## 5. 最终预测与验证
训练结束后，代码抽取了测试集的前 10 张图片输入模型。

通过 torch.max(..., 1) 找到概率最大的类别作为预测值。

最后同时打印出预测数字和真实数字，供肉眼直观比对模型的分类效果。

## 🔬 实验 1：修改训练轮数（EPOCH）

`EPOCH` 决定了模型把**整个训练集**完整学习多少遍。

### ✍️ 修改指南

* **增加轮数（推荐）：** 将 `EPOCH = 1` 改为 `EPOCH = 5` 或 `EPOCH = 10`。
* **减少轮数：** 当前已经是 1，无法再减小。

### 🎯 观察重点

* **准确率趋势：** 随着 Epoch 的增加，测试准确率（test accuracy）是否在持续上升并最终趋于平稳？
* **训练损失：** 训练损失（train loss）是否在逐步下降？
* **副作用：** 如果 Epoch 加得太大（比如 50），是否会出现“训练集损失一直在降，但测试集准确率不再提升甚至下降”的过拟合（Overfitting）现象？

---

## 🔬 实验 2：修改学习率（LR / Learning Rate）

`LR` 决定了模型参数在每次更新时“步子迈得有多大”。

### ✍️ 修改指南

* **尝试极端的学习率，观察两极分化的结果：**
* **调大（步子过大）：** 将 `LR = 0.001` 改为 `LR = 0.1` 甚至 `LR = 0.5`。
* **调小（步子过小）：** 将 `LR = 0.001` 改为 `LR = 0.00001`。



### 🎯 观察重点

* **当 LR 太大时（0.1）：** 你可能会发现 `train loss` 变成了一个非常大的数字（甚至变成 `nan`），测试准确率一直在 10% 左右（相当于随机瞎猜）徘徊。这是因为步子太大，直接“迈过了”最优解。
* **当 LR 太小时（0.00001）：** 训练损失下降得极度缓慢，训练完 1 个 Epoch 后，准确率几乎没有提升。
* **最佳实践：** 寻找能让 Loss 下降最快、最稳的那个“黄金折中值”。

---

## 🔬 实验 3：修改批大小（BATCH_SIZE）

`BATCH_SIZE` 决定了模型每次看多少张图片后，才更新一次网络参数。

### ✍️ 修改指南

* **调大 BATCH_SIZE：** 将 `BATCH_SIZE = 50` 改为 `BATCH_SIZE = 256` 或 `BATCH_SIZE = 1024`。
* **调小 BATCH_SIZE：** 将 `BATCH_SIZE = 50` 改为 `BATCH_SIZE = 4` 或 `BATCH_SIZE = 1`。

### 🎯 观察重点

* **打印频率与速度：** * 当 `BATCH_SIZE` 变大时，代码中 `if step % 50 == 0:` 的打印信息出现得变慢了（因为总 Step 变少了），但整个 Epoch 运行的总时间可能会变快。
* 当 `BATCH_SIZE` 极小时（如 1），屏幕会疯狂刷屏，训练速度明显变慢。


* **震荡幅度：** 较小的 BATCH_SIZE 会让每次打印的 `train loss` 跳跃很大（不稳定），而较大的 BATCH_SIZE 会让 Loss 下降显得更平滑，但如果太大，模型最终的泛化能力可能会略微下降。


```python
import torch
import torch.nn as nn

def train(model, train_loader, optimizer, criterion, device):
    model.train()

    total_loss = 0

    for x, y in train_loader:
        x, y = x.to(device), y.to(device)

        # forward
        outputs = model(x)
        loss = criterion(outputs, y)

        # backward
        optimizer.zero_grad()
        loss.backward()
        optimizer.step()

        total_loss += loss.item()

    return total_loss / len(train_loader)
```

```python
import torch

def test(model, test_loader, device):
    model.eval()
    correct = 0
    total = 0

    with torch.no_grad():
        for x, y in test_loader:
            x, y = x.to(device), y.to(device)

            outputs = model(x)
            _, predicted = torch.max(outputs, 1)

            total += y.size(0)
            correct += (predicted == y).sum().item()

    acc = correct / total
    return acc
```

# Step 5：Summarize（总结）

能够理解：

---

上面的多层感知机（MLP）实验，你会发现 MLP 在处理 28x28 的小手写数字时还能应付，但面对更复杂的视觉任务就显得力不从心了，例如图像如果是彩色的RGB三个通道的 1920 X 1080 像素的图片，展开后维度有多长？ 所以这时候CNN卷积神经网络就是处理图像的更好的模型了。

下面拆解为什么卷积神经网络（CNN）是更好的选择：

---

## 1. 为什么 CNN 比 MLP 好？

在MLP代码中，我们把二维图像硬生生拉成了一条一维的“面条”（784 维向量）。这种做法面对图像时有两个致命缺陷：

* **内容结构的破坏：** 图像的本质在于像素之间的**空间相对位置**。比如，猫的眼睛和鼻子在空间上是相邻的。MLP 把图像拉直后，第 1 行的最后一个像素和第 2 行的第一个像素在向量里变成了邻居，这彻底破坏了图像的几何拓扑结构。
* **维度灾难（参数爆炸）：** 原代码中仅仅一个全连接层 `nn.Linear(784, 784)` 就产生了 $784 \times 784 \approx 61.4$ 万个权重参数。如果处理一张普通的彩色高清图片（如 1080p：$1920 \times 1080 \times 3$），单层全连接的参数量将直接飙升到 **6.2 万亿**个！这不仅会让显存瞬间崩塌，还会导致极其严重的**过拟合**。

CNN 通过引入局部连接和数学上的卷积操作，完美解决了上述问题，在处理高维空间数据时，其准确率、参数量和泛化能力都完爆 MLP。

---

## 2. CNN 做了什么改进？

CNN 的核心进化可以总结为以下三个关键机制：

### 🔍 特征提取 (Feature Extraction)

* **MLP 的做法：** 让每一个神经元同时去看整张图的所有像素，导致重点不突出。
* **CNN 的改进：** 采用**分层特征提取**。网络底层的卷积核（Kernel）只负责看局部，提取最基础的边缘、线条和色彩渐变；随着网络加深，高层的卷积核将这些基础线条融合成局部器官（如眼睛、车轮）；最终层则组合出高阶的抽象语义（如整只猫、整辆车）。这种由浅入深的提取方式，极大地提升了模型的表达能力。

### 🌐 空间结构 (Spatial Structure)

* **MLP 的做法：** 丢弃维度，将 2D/3D 数据降维成 1D 向量。
* **CNN 的改进：** 卷积核直接在图像的二维（宽、高）甚至三维（加上色彩通道）空间上滑动。它保留了像素之间的**空间局部相关性**。不仅如此，通过卷积和池化（Pooling）的组合，CNN 具备了**平移不变性**——无论那只猫在图片的左上角还是右下角，卷积核都能通过滑动检测到它，而 MLP 面对位置偏移则需要重新学习。

### ✂️ 参数共享 (Parameter Sharing)

* **MLP 的做法：** 每个输入像素和每个输出神经元之间都有一个独立的权重（全连接）。
* **CNN 的改进：** 引入了两招：
1. **局部连接（Local Connectivity）：** 神经元不再和整张图连接，而是只连接一个局部区域（比如 $3 \times 3$ 的窗口）。
2. **参数共享（Parameter Sharing）：** 同一个卷积核（也就是一组权重）在整张图片上从左到右、从上到下**滑窗共享**。这意味着，“负责检测垂直边缘”的 $3 \times 3$ 滤波器在图片左上角和右下角用的是同一套参数。


> **直观对比：** 同样处理一张 $1000 \times 1000$ 的单色图片，输出 1000 个特征。MLP 需要 $10^9$ 个参数；而 CNN 若使用 1000 个 $3 \times 3$ 的卷积核，参数量仅为 $3 \times 3 \times 1000 = 9000$ 个！参数量直接缩减了 10 万倍。

---

## 3. 什么时候用 CNN？

CNN 的本质是处理**网格化（Grid-like）且局部有高度相关性**的数据。只要数据满足“局部像素/点彼此亲密，整体具备平移规律”，就是 CNN 的主场：

### 🖼️ 图像 (Image)

这是 CNN 的大本营，几乎统治了所有计算机视觉（CV）任务。

* **典型应用：** 图像分类（如人脸识别）、目标检测（如自动驾驶识别行人路标）、图像分割（如医疗影像中提取肿瘤区域）。

### 🗣️ 语音与音频 (Audio)

声音虽然是一维的时间信号，但它的分析离不开 CNN。

* **典型应用：** 在语音识别中，通常会将音频经过傅里叶变换，转化为**梅尔倒谱系数（MFCC）时频图（Spectrogram）**。这张图的横轴是时间，纵轴是频率，它本质上就是一张“声音的图像”，此时使用 2D CNN 可以极好地捕获语音中的声学特征和共振峰。

### 🗺️ 空间数据 (Spatial Data)

任何表现为 1D、2D 或 3D 网格，且空间位置至关重要的数据。

* **典型应用：** * **气象与地理：** 卫星遥感地图分析、气温/降水空间分布预测。
* **视频分析：** 视频可以看作是“2D图像 + 1D时间轴”的 3D 网格数据，使用 3D CNN 可以同时捕捉画面空间和时间前后的动作变化。
* **文本与序列（1D CNN）：** 虽然如今 NLP 多用 Transformer，但在文本分类或 DNA 序列分析中，1D CNN 经常被用来通过滑动窗口快速提取局部短语（N-gram）特征。
